# 使用指南 - 防止 Agent 偷懒（巡检设置 / Agent 鞭策）

‍

> Mobius 后台**每 60 秒巡检一次**：如果发现某个会话的 Agent 已经停工、但 `running.flag` 还没删除（典型表现：卡住、偷懒、忘了收尾），就自动往该会话发一条消息**鞭策它继续干**。每个项目可在「巡检设置 - Agent鞭策设置」里自定义提醒文案与提醒节奏。

### 1. 找到设置入口

打开项目，在右侧**项目设置**里找到「**巡检设置 - Agent 鞭策设置**」卡片。

### 2. 配置提醒文案与节奏

![image](https://serve.nutshellai.cn/publish/auto/tutorial/06_patrol_nudge.jpg)

- **① 提醒消息**：Agent 偷懒时自动发到会话里的文案。写清楚「请继续完成任务并删除 running.flag」之类即可。
- **② Issue 会话策略**：普通任务会话的三档参数 —— `Init`（首次等待分钟）、`Backoff`（退避倍数）、`Patience`（最多提醒次数）。
- **③ Research 会话策略**：研究任务会话的三档参数，默认比 Issue 更宽松（研究本身耗时长）。
- **④ 计算公式**：第 N 次提醒后，下一次的等待时间 = `Init × Backoff^N`；达到 `Patience` 次后就**只记日志、不再打扰**。

> 默认值：Issue `10 / 2 / 3`（即 10 分钟后第一次提醒，之后 20、40 分钟……最多 3 次）；Research `30 / 5 / 5`。改完会自动保存到当前项目。
