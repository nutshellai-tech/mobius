# 视频封面制作拓展设计

日期：2026-07-12

## 背景

`video-cover` 拓展目前仍是脚手架：前端只有调试备注保存，后端只有 `whoami/save_note/load_note`。本次首版目标是把它做成真实可用的视频封面制作工具。

用户已确认首版范围：

- 支持上传视频和参考封面图。
- 支持填写封面文案和平台。
- 后端调用生图接口生成封面。
- 输出横屏和竖屏封面。
- 前端可预览和下载结果。
- 同时支持 AI 直出文字和“文字准确模式”。
- 不做视频片头合成，不输出带封面 mp4。

## 产品目标

首版是一个封面制作工作台，而不是宣传页。用户打开拓展后可以完成一条完整链路：

1. 上传视频或参考图。
2. 输入标题、副标题和补充风格要求。
3. 选择平台、比例和文字模式。
4. 创建生成任务。
5. 在页面里查看进度。
6. 预览生成的横屏/竖屏封面。
7. 下载成品或重试失败项。

## 非目标

首版明确不做以下内容：

- 不把封面拼入视频片头。
- 不分析视频内容自动提取标题。
- 不做多人协作或团队素材库。
- 不做复杂的封面编辑器，如拖拽排版、图层编辑、局部重绘。
- 不绕过平台上传限制，视频上传沿用现有拓展上传接口的 50MB 限制。

## 用户界面

页面采用两栏工作台布局。

左侧是输入和任务配置：

- 视频上传：接受常见视频文件，作为任务素材和后续扩展入口。首版不读取完整视频内容。
- 参考图上传：可上传 0 到多张封面参考图，用于影响风格。
- 文案输入：主标题、副标题、补充说明。
- 平台选择：小红书、B站、抖音/视频号、通用。
- 比例选择：横屏 `16:9`、竖屏 `3:4`、竖屏 `9:16`，支持多选。
- 文字模式：默认 `AI 直出`，可切换到 `文字准确模式`。
- 生成按钮：创建异步任务。

右侧是结果和历史：

- 当前任务状态：`queued`、`running`、`done`、`failed`。
- 结果网格：按比例展示封面预览、状态、下载按钮、重试按钮。
- 最近任务历史：展示最近任务，点击后恢复详情和下载入口。
- 配置状态提示：生图配置缺失时禁用生成并显示原因。

移动端改为单列：配置在上，结果在下。所有按钮、预览卡片和状态文案需要避免重叠。

## 文字模式

### AI 直出

模型直接生成带中文标题和副标题的封面。优点是字体、光影和背景融合最好；缺点是中文偶发错字或糊字。页面需提示用户人工核对，并提供“用文字准确模式重做”的操作。

### 文字准确模式

模型生成无字背景，后端使用 PIL 与 NotoSansCJK 字体叠加主标题、副标题和角标。优点是文字 100% 准确；缺点是视觉融合和排版变化不如 AI 直出。首版只需要提供稳健排版，不做手动调字。

## 平台和比例

平台会影响 prompt 和默认画面风格：

- 小红书：明亮、强标题、强点击感、适合 3:4 和 16:9。
- B站：信息清晰、标题醒目、可带教程/干货感，适合 16:9。
- 抖音/视频号：竖屏优先、主体更大、标题更短，适合 9:16。
- 通用：不过度平台化，适合复用到多个渠道。

比例会影响输出文件和 prompt：

- `16:9`：横屏封面。
- `3:4`：小红书竖图封面。
- `9:16`：短视频竖屏封面。

每个任务可以选择多个比例。每个比例独立生成、独立校验、独立失败和重试。

## 后端架构

拓展后端仍使用 `mobius/extension/video-cover/backend/extension_backend_handler.js`，保持 CommonJS、stateless，不在模块顶层持久化连接或定时器。所有数据写入 `ext_data_dir` 下。

建议数据结构：

```text
ext_data_dir/
  users/<username>/
    state.json
    uploads/
      <platform-uploaded-files>
    jobs/
      <job_id>/
        job.json
        inputs.json
        results/
          cover-16x9.png
          cover-3x4.png
          cover-9x16.png
          bg-16x9.png
        logs/
          worker.log
```

`state.json` 存最近上传和最近任务索引。`job.json` 是任务状态真相源，worker 每完成一个比例就更新一次。

## 后端动作

`get_state`

返回用户身份、生成配置状态、最近上传、最近任务列表。

`register_upload`

前端通过 `/api/extensions/video-cover/upload` 上传文件后，将返回的文件信息交给 handler 记录。handler 校验文件路径必须在当前拓展的用户上传目录下。

`create_job`

校验标题、平台、比例、文字模式和参考图路径。创建任务目录，写入 `job.json` 和 `inputs.json`，启动独立 worker，然后立即返回 `job_id`。

`get_job`

返回单个任务的当前状态、结果列表、错误信息和资产 ID。任务详情不直接内嵌图片 base64，避免多张图把 handler JSON 响应撑大。

`list_jobs`

返回最近任务摘要，用于历史区。

`retry_item`

对某个失败或不满意的比例重新生成。复用原任务参数，只重跑对应比例并更新结果。

`delete_job`

删除任务目录和状态索引中的记录。

`get_asset`

读取当前用户某个任务结果图，返回单张 PNG 的 `data_url`、文件名、大小和 mime type。handler 必须校验资产路径在当前用户任务目录下，并拒绝返回超过 4MB 的单图，避免触碰 handler 5MB JSON 响应上限。前端用 `data_url` 做预览，并转换为 blob 触发下载。

## 异步任务模型

拓展 handler 单次请求上限是 30 秒，而单张生图通常接近 25 秒。横屏和竖屏多图同步生成会超过上限，所以首版必须采用异步任务。

`create_job` 只负责入队和启动 worker。worker 使用独立 Node 进程执行，写入同一个任务目录。前端每 2 到 3 秒轮询 `get_job`，直到所有比例完成或失败。

如果 worker 启动失败，任务状态标为 `failed`。如果单个比例失败，任务整体可以是 `partial` 或 `done_with_errors`，前端仍展示成功的结果。

## 生图流程

1. 读取生图配置，优先复用 `mobius-auto-recording` 的 `cover_config.json`，也支持环境变量。
2. 根据平台、比例、文字模式、标题、副标题和参考图构造 prompt。
3. 对每个比例独立调用生图接口。
4. 写入临时文件。
5. 使用封面质量校验拦截纯色块、空白图或无法解码图。
6. 校验通过后 rename 为正式结果文件。
7. 更新 `job.json` 中对应比例的状态和 URL。

质量校验沿用 `video-cover-design` skill 的策略：用 ffmpeg 缩放到 `64x36`，输出 raw rgb24，统计量化颜色数；低于阈值视为无效封面。

## 静态访问和下载

生成物需要前端能预览和下载。拓展静态路由默认只服务 `frontend/dist`，不直接暴露 `ext_data_dir`，所以首版不依赖静态 URL。

首版使用 `get_asset` 按需读取单张结果图：

- `get_job` 返回结果元数据和 `asset_id`。
- 前端在结果卡片进入可见区域或用户点开时调用 `get_asset`。
- handler 只返回单张图片的 `data_url`，避免一次返回多张图超过 JSON 上限。
- 单张结果图超过 4MB 时，前端显示“图片过大，需重新生成或后续使用下载路由”。
- 前端把 `data_url` 转为 blob URL 用于预览和下载。

如果后续要支持大批量下载或更大的资产，再考虑给拓展系统增加受控的 `ext_data_dir` 文件下载路由。

下载按钮应直接下载 PNG，文件名包含任务标题、平台、比例和时间，例如：

```text
视频封面-小红书-16x9-20260712.png
```

## 错误处理

- 生图配置缺失：`get_state` 返回 `image_config_ok=false`，前端禁用生成按钮。
- 上传文件过大：平台上传接口返回错误，前端直接展示。
- 无标题：前端和后端都拦截。
- 未选择比例：前端和后端都拦截。
- 生图接口失败：对应比例标记失败，保留错误摘要。
- 质量校验失败：对应比例标记失败，不展示坏图为成功结果。
- AI 中文错字：不自动判定，前端提供用文字准确模式重做。

## 测试计划

后端测试：

- `get_state` 能返回配置状态和空历史。
- `register_upload` 只接受当前用户上传目录下的路径。
- `create_job` 对非法平台、空标题、空比例做校验。
- 任务目录和 `job.json` 能正确创建。
- 质量校验能识别有效封面和纯色块。

前端验证：

- 上传视频和参考图后能显示在输入区。
- 创建任务后能轮询状态。
- 多比例结果能独立显示成功或失败。
- 完成结果能预览和下载。
- 历史任务刷新后仍可打开。
- 桌面和移动端布局无明显重叠。

端到端验证：

- 使用真实生图配置生成至少一个 `16:9` 和一个竖屏封面。
- 分别验证 AI 直出和文字准确模式。
- 用 `display_images` 展示生成结果给用户确认。

## 后续扩展

首版完成后可追加：

- 平台模板库和品牌色预设。
- 手动微调标题位置、字号和徽章。
- 与 `mobius-auto-recording` 的双封面体系联动。
