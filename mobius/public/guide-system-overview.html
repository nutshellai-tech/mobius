一、场景级首触引导（3 条）

首次进入某功能区时按用户×场景维度自动触发，看完或跳过标记 seen（跨设备不再弹）。可在帮助中心「重温」重看。

管理中心场景首触
触发：管理员首次打开管理中心 overlay（非路由，靠 admin-center-header 元素探测）　·　selector 体系：data-tour="admin-*"
管理中心 overlay→逐 tab 切换讲解→收尾
管理中心 overlay（用户菜单 → 管理中心）
  • admin-center-header这里是管理中心：系统级管理总入口，八个模块按需切换
  • admin-tab-bar按模块切换：这里八个模块各自独立
  • admin-section-users用户管理：管员工账号、角色、权限（默认 tab，直接讲）
  • admin-tab-models → admin-section-models模型接入：配置 AI 模型通道、密钥、代理
  • admin-tab-settings → admin-section-settings系统设置：模型创建配额、全局默认模型、代理链
  • admin-tab-extensions → admin-section-extensions拓展管理：安装、启用、隐藏莫比乌斯拓展插件
  • admin-tab-migration → skill-memory-manage-panelSkill与Memory管理：迁移并管理项目级技能和记忆
  • admin-center-header改动影响所有人：全站生效，改前先确认；重看从右上角问号
Research 课题页场景首触
触发：路由 /u/:user/p/:project/r/:research（靠 research-header 元素探测）　·　selector 体系：data-tour="research-*"
研究课题工作台→?view=blackboard→?view=graph→收尾
Research 课题页 /u/:user/p/:project/r/:research
  • research-header研究课题工作台：莫比乌斯的研究子系统，可组建多智能体团队
  • research-agent-list研究员列表：首席锁定不可删统筹全局，助理并行写回黑板
  • research-new-agent新建研究员：可创建单个，也可一键拉起团队
  • research-toggle-blackboard → research-blackboard协作黑板：汇总各研究员实时写回的进展与结论，自动刷新
  • research-toggle-graph → research-graph研究图谱：节点图展示团队分工与进展，可拖拽布局
  • research-header研究是项目级能力：需在项目设置开启；重看从引导中心
self-cognition 工作台场景首触 · 独立 driver
触发：访问 /extension/self-cognition/（扩展页，独立 vanilla JS + esm.sh 加载 driver.js）　·　selector：纯 CSS id
#hero 研究雷达→#heroFeed→#paper-section→#implementation-section→#evolution-section→闭环
self-cognition 扩展页 /extension/self-cognition/
  • #hero研究雷达：莫比乌斯研究自己的工作台，把研究做成持续流程
  • #heroFeed卡片流：汇总论文、产品、启发点
  • #paper-section论文区：列出待审论文，可打开论文工作台选论文、挑启发
  • #implementation-section启发落地区：每条启发可推进成自迭代任务，研究到落地的闭环
  • #evolution-section自进化历史：已落地改动按时间线排列
  • #hero一条闭环：论文→启发→落地→历史，自我改进完整闭环

四、各 Demo 路线差异

创建演示项目（birthday）demo
从首页起步，全程手动创建项目→任务→会话。session-chat 走 extension 全套日志卡（birthday 与 extension 共用 segment）。
首页→项目表单→项目页→任务单→会话→执行→完成→清理
走全套公共骨架，session-chat 含 8 张日志卡（user/event/reasoning/tool-call/tool-result/token-count/assistant×2）。
配置开发资料（context-setup）demo
在项目页分三阶段：下载+上传项目知识 → 下载+上传项目方法 → 建验证任务。可选远程算力弹窗。
首页→项目表单→项目页·知识阶段→项目页·方法阶段→项目页·建任务→任务单→会话→执行→完成→清理
项目页·context 专属高亮（/u/:user/p/:project）
  • project-settings-panel先准备真实资料
  • project-context-memory-download下载项目知识
  • project-context-skill-download下载方法文件
  • project-memory-manager / project-memory-upload-knowledge上传项目知识
  • project-skill-manager / project-skill-upload-file上传项目方法
  • project-context-whitelist个人资料筛选
远程算力弹窗（可选，remote-compute-modal）
  • remote-compute-row / -test / -hardware选机器、测连通、探硬件
  • remote-compute-memory-preview / -create-memory预览并创建项目记忆
导入已有代码（project-import）demo
在项目页分三阶段：下载上传样例 → 打开 VSCode 确认上传 → 清空样例 → 建导入任务（带 gitUrl）。
首页→项目表单→项目页·下载样例→项目页·上传→项目页·清空→建导入任务→执行→清理
项目页·import 专属高亮（/u/:user/p/:project）
  • project-import-sample-download下载真实上传样例
  • vscode-open-button打开网页代码编辑器
  • project-import-confirm-upload-sample确认上传完成
  • project-import-clear-upload-sample清空样例，继续下一种方式
拓展演示（extension）demo
与 birthday 同骨架。预览页含 session-preview-mobius-extension-skill + session-preview-logo-memory 两张专属卡。
首页→项目表单→项目页→任务单→会话预览(含拓展技能卡)→执行→完成→清理
会话预览页·extension 专属（session-preview）
  • session-preview-mobius-extension-skill保留拓展技能：mobius-extension 协议与边界
  • session-preview-logo-memory保留项目知识：光点标志空间案例约束
莫比乌斯自迭代（self-evolve）demo
直接进入已有自迭代项目（不新建），任务只改一处演示时间文字。预览页含 3 张专属记忆卡。不走清理（自迭代项目不删）。
自迭代项目页→任务单→会话预览(3张记忆卡)→执行→完成
会话预览页·self-evolve 专属（session-preview）
  • session-preview-self-evolve-required-memory保留项目知识：莫比乌斯自主开发项目知识
  • session-preview-self-evolve-project-memory保留实现记录：引导系统与已有实现位置
  • session-preview-self-evolve-guide-memory保留文案规则：中文优先、短句、少讲实现
