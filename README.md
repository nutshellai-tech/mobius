## 🌀 Mobius 是什么

> Mobius 不是普通 AI 助手，而是一个连接人类、智能体、机器、知识、算力和物理终端的**自进化超级智能体系统**，让复杂目标能够在数字世界与物理世界之间持续协作、持续沉淀、持续生长。

Mobius 以**自进化 Agent OS** 为内核，把人类团队、Agent 编队、知识资产、算力集群和物理智能终端组织起来，让数字系统、硬件设备、机器人与现实任务在**同一个协作闭环**中持续开发、部署、学习和进化。

```text
普通 AI 助手：
    输入 → 模型 → 输出 → 结束

Mobius：
    想法 → 拆解 → 多智能体并行 → 验证 → 交付 → 沉淀 ↻ 系统继续生长
```

Mobius 自己也用 Mobius 开发 Mobius。它不只是在完成任务，也在用任务反过来塑造自己。

## 🌍 时代问题：谁来定义系统？

大模型让执行能力变得越来越便宜：写代码、生成页面、复刻工具、调用接口、整理资料，都会越来越容易。真正稀缺的，不再只是某一个功能或某一个界面，而是**系统如何被组织、被操作、被协同、被持续进化**。

今天的 Claude Code、Codex、Cursor 等工具已经显著提升了开发者效率，但它们仍然更接近**开发者工具**。上下文要自己组织，文件要自己管理，Memory、Skill、项目资料和执行状态往往散落在文件系统、命令行和开发环境里。懂代码的人可以串联起来；但对普通用户、团队管理者和非技术员工来说，门槛仍然很高。

这正是 Mobius 要回答的问题：

> **AI 时代，谁来定义系统？**
> 过去是开发者定义系统，用户适应系统；**未来应该是用户提出目标，Agent 改造系统，系统与用户共同进化。**

## 🔍 与现有工具的差异

Mobius 不只是某一个能力的强化，而是把原本散落的 Agent 操作能力提升为一个**可交互、可协作、可沉淀、可持续进化**的系统层。

| 对比维度 | 现有工具 | Mobius |
|---|---|---|
| **代码 Agent**（Claude Code / Codex） | 在代码层面执行任务 | 把代码 Agent 纳入更大协作网络：项目、团队、设备、产品孵化的一部分 |
| **开发增强**（Cursor） | 服务开发者工作流 | 把 Memory、Skill、上下文、Deep Research 提升为图形化、可协作、可沉淀的系统层，让非技术用户也能参与 |
| **机器人框架**（具身智能） | 偏向单一物理执行体 | 通过项目、代码、算力、知识、终端协作，持续开发、部署和增强机器人能力 |
| **团队协作系统** | 记录任务、文档和沟通 | 让人、Agent、机器和算力进入**同一条生产线**，自动交接、执行、沉淀、进化 |
| **操作系统** | 强调资源管理和应用运行 | 组织**数字智能与物理世界**的协作，让真实任务在系统、设备、机器人和算力之间持续流动 |

## 🌐 人机物智算协同

Mobius 的"触手，无处不达"具体落到现实世界的协同对象上。五个原本散落的能力节点，在 Mobius 里被组织成一张可持续运转的协同网络：

| 节点 | 含义 | 在 Mobius 中的角色 |
|---|---|---|
| 👤 **人** | 个人 · 团队 · 组织 | 提出目标、协同开发、验收反馈（通过小莫自然语言交互） |
| 🧠 **智** | Agent 编队 · AI 模型 · 知识资产 | 拆解 · 推理 · 执行 · 沉淀（多智能体工作面） |
| 💻 **算** | GPU 集群 · 云端 · 本地算力 | 训练 · 仿真 · 测试 · 数据处理（SSH / 任务分发） |
| 🤖 **机** | 机器人 · 工作站 · 终端设备 | 开发 · 调试 · 部署 · 现实动作（AIMUX / 远程终端） |
| 🔌 **物** | 嵌入式 · 传感器 · 执行器 | ESP32 · 树莓派 · Jetson · 现实对象（烧录 / 运行 / 观测） |

五个节点通过 **Mobius Core**（自进化超级智能体系统内核）形成自进化回路：

```text
                    ┌─────────────┐
                    │     智      │
                    │ Intelligence│
                    └──────┬──────┘
                           │
   ┌───────────┐    ┌──────▼──────┐    ┌───────────┐
   │    人     │◀──▶│   Mobius    │◀──▶│    算     │
   │   Human   │    │    Core     │    │  Compute  │
   └─────┬─────┘    └──────┬──────┘    └─────┬─────┘
         │                 │                 │
         │           ┌─────┴─────┐           │
         │           │           │           │
   ┌─────▼─────┐              ┌──▼──────────┐
   │    物     │              │     机      │
   │  Things   │              │   Machine   │
   └───────────┘              └─────────────┘

           执行 → 反馈 → 沉淀 → 新能力 ↻
```

## ⚡ 四个核心关键词

| 关键词 | 含义 |
|---|---|
| 🤝 **协同** | Mobius 的核心能力不是单个智能体变强，而是让人类团队、Agent 编队、知识资产、算力资源和物理终端**共同完成更复杂的目标**。 |
| 🌱 **自进化** | 每一次任务执行、项目交付、设备部署、用户反馈和知识沉淀，都会反过来成为系统继续生长的材料。 |
| 🥚 **自孵化** | Mobius 产出的不是一次性回答，而是可以继续运行、继续迭代、继续扩展的工具、产品、研究成果和机器人能力。 |
| 🔁 **共同进化** | Mobius 不只是系统自己升级，而是**人与 Agent、系统与用户、组织与知识资产共同进化**。系统越用越理解用户，用户也越用越会驾驭 Agent。 |

## ✨ 七大亮点

> 这不是七个互相分散的功能点，而是一条从"自生长系统"到"团队和科研生产力"的完整逻辑链。

<details open>
<summary><b>01 · 自迭代，自生长</b> &nbsp;—— 绝对亮点</summary>

Mobius 的第一亮点是自迭代。它可以把自己的源码作为项目来管理：用户提出改进点，系统创建 Issue 和 Session，Agent 进入真实代码目录修改、测试、提交，最后把改进带回 Mobius 自身。

```text
需求 → Issue → Session → Agent 改代码 → 测试 → 部署 → Mobius 变强 ↻
```

每次修复、每次文档补充、每次技能沉淀，都可能成为下一次执行的上下文、Memory、Skill 或扩展能力。**自迭代不是附属能力，而是 Mobius 的第一性卖点。**

</details>

<details open>
<summary><b>02 · 集群思考协作，从点到面</b></summary>

一个想法进入 Mobius，不再只是一个 Agent 给出一段回答，而是可以展开成一张**智能体工作面**：不同 Agent 负责不同子任务，不同 Session 承载不同执行链路，黑板和研究图谱把中间信息持续汇合。

```text
一个点状需求
    ├─ Agent A：调研
    ├─ Agent B：实现
    ├─ Agent C：验证
    ├─ Agent D：文档
    └─ Chief / Blackboard：收敛和沉淀
```

</details>

<details open>
<summary><b>03 · 团队协作 max</b></summary>

第二点偏 AI 内部结构，第三点偏人类组织结构。Mobius 要解决的不只是"一个人怎么调 Agent"，还包括**团队成员、管理员、项目负责人、AI Session 和代码工作区如何在同一张作战图里协同**。

```text
一个人使用 Mobius：一人成群
多人使用 Mobius：人类团队 + Agent 编队协同作战
管理者使用 Mobius：鸟瞰项目、成员、Session 和交付状态
```

</details>

<details open>
<summary><b>04 · 深度科研系统</b></summary>

深度科研系统是 Mobius 的旗舰场景。它把多智能体协作、黑板系统、研究图谱、远程算力和结果沉淀放进一条科研链路里：从一个想法出发，拆成多个研究子问题，调动多个 Agent 做文献、实验、复现、证伪、总结，必要时调度远程 GPU 或异构算力节点。

```text
想法 → 文献扫描 → 假设拆解 → 多 Agent 实验
     → GPU/远程算力协同 → 证实/证伪 → 图表/论文/报告
```

</details>

<details open>
<summary><b>05 · 灵活的拓展系统</b></summary>

Mobius 不应该被固定成一个单一 SaaS。它的扩展系统允许每个特殊能力都长成独立应用：**独立前端、独立后端、独立数据目录、独立构建和调度流程**。官网、科研工具、PPT 生成器、财经新闻墙、JSON 可视化、小游戏、三体仿真，都可以以扩展形式存在。

```text
一句话想法 → 新扩展 → 独立页面 → 独立后端 → 可复用应用
```

</details>

<details open>
<summary><b>06 · 智能小莫</b></summary>

小莫是 Mobius 的自然交互层。用户不需要理解项目、Issue、Session、Skill、Memory、扩展、模型渠道这些底层结构，很多时候只需要用自然语言说出目标。小莫负责理解当前页面、当前项目、当前任务上下文，并把请求转成可执行动作或可确认选项。

```text
用户一句话
  → 小莫理解页面和上下文
  → 创建任务 / 查找项目 / 配置入口 / 提醒确认
  → 后台系统执行
```

**后台可以很复杂，前台必须自然。**

</details>

<details open>
<summary><b>07 · 任意模型，一键接入</b></summary>

模型不是 Mobius 的边界，工作流才是。Mobius 可以接 GLM、Claude、GPT、Codex、本地模型、OpenAI 兼容接口、Anthropic 兼容接口，以及未来更多模型渠道。模型可以替换，但**项目、上下文、任务、团队协作和交付流程**保持在同一套系统里。

```text
GLM / GPT / Claude / Codex / Local Model / Future Model
                     │
                     ▼
        Mobius 项目、任务、上下文和交付体系
```

</details>

## 🧱 工作模型

Mobius 的基本生产单元：

```text
┌──────────┐    ┌──────────┐    ┌──────────┐
│ PROJECT  │ ─▶ │  ISSUE   │ ─▶ │ SESSION  │
│ 项目上下文│    │ 目标和验收│    │ 一次真实 │
│ 容器     │    │ 标准     │    │ Agent 执行│
└──────────┘    └──────────┘    └──────────┘
     ▲                                │
     │      ┌──────────────┐          │
     └──────│  沉淀回流     │◀─────────┘
            └──────────────┘
```

| 层级 | 作用 |
|---|---|
| **Project** | 保存项目背景、目录绑定、Memory、Skill、Research 配置 |
| **Issue** | 表达一个可验收目标，承载任务需求和讨论 |
| **Session** | 一次真实 Agent 执行，有快照、有输出、有状态 |
| **Extension** | 可独立运行的小应用，把新能力产品化 |
| **Xiao Mo（小莫）** | 面向用户的自然语言入口，把复杂系统变成一句话交互 |

## 💎 商业与生态价值

Mobius 的价值不只在代码本身，而在围绕 Agent 运行所需的**完整服务网络**：模型调用、语音输入输出、多端交互、插件系统、远程访问、端口代理、部署环境、任务运行基础设施和算力协同。

即使系统开放出去，完整体验仍然依赖持续服务。大模型 Token、ASR、TTS、插件市场、远程运行环境、内网穿透、任务调度和外部 API，都会构成 Mobius 长期服务能力的一部分。

- **对个人**：Mobius 是一个越用越像自己的智能系统。用户可以先提出一个很小的需求，Mobius 在一次次交互中沉淀习惯、偏好、流程和结果，逐渐把它们变成系统能力。它不是一个被写死的产品，而是一块可以持续塑形的智能系统。
- **对企业**：Mobius 是一套人智协同的生产系统。它降低专业门槛，让非顶级技术员工也能借助 Agent 完成更复杂的工作；它提升交付效率，让任务拆解、执行、反馈、测试、汇报和验收进入长程闭环；它沉淀组织知识，让项目经验、Memory、Skill、插件、执行记录和协同方式成为可复用资产。

> **Mobius 重新定义 AI 时代的系统范式**：过去是开发者定义系统，用户适应系统；未来是用户提出目标，Agent 改造系统，系统与用户共同进化。

## 🚀 快速开始

#### 方式一：容器中安装和运行（所有操作系统，推荐）

```bash
# 1. 克隆仓库
git clone https://github.com/nutshellai-tech/mobius.git
cd mobius

# 2. 生成配置（随机秘钥密码，可以手动配置跳过此步）
python3 conf_prepare.py --docker && python3 conf_check.py --docker

# 3. 构建 base 镜像（仅环境，不含代码）
docker build -t imac-mobius-base:latest -f deploy/Dockerfile . && docker build -t imac-mobius-exe:latest .

# 4. 启动
docker compose up
```

#### 方式二：直接部署（Linux or MacOS）

```bash
# 1. 安装 tmux git 等必要依赖
sudo apt install tmux python3 git curl proxychains openssh-server build-essential

# 2. 安装claude code和codex（其中之一即可，但建议都安装）
npm install -g @anthropic-ai/claude-code @openai/codex

# 3. 克隆仓库
git clone https://github.com/nutshellai-tech/mobius.git
cd mobius

# 4. 生成和配置秘钥（将会复制 .env.default 到 .env，并创建随机密码）
python3 conf_prepare.py && python3 conf_check.py

# 5. 安装项目依赖环境 （前端+后端）
cd ./mobius && npm install && cd ./frontend && npm install && cd ../..

# 6. 运行
python3 start.py
```
